API Project

Desarrollo de un API Rest utilizando buenas prácticas de desarrollo

Proyecto educativo para aprender las mejores prácticas en el desarrollo de APIs con TypeScript, Express, ESLint y Prettier. Cada paso está documentado y explicado para facilitar el aprendizaje.

Creación del primer endpoint de la API con configuración completa del proyecto

TypeScript ESLint Prettier Express

Creación del Proyecto

1. Inicialización del Proyecto

Comenzamos creando un nuevo proyecto Node.js:

Terminal 
npm init -y

2. Instalación de Dependencias

Instalamos Express como dependencia principal:

Terminal 
npm i express

Instalamos las dependencias de desarrollo para TypeScript, ESLint y Prettier:

Terminal 
npm i -D typescript ts-node tsconfig-paths eslint prettier @types/node @types/express @typescript-eslint/eslint-plugin @typescript-eslint/parser

Configuración del Proyecto

3. Archivo src/app.ts

Creamos el archivo principal de la aplicación:

src/app.ts 
import express from 'express';

const app = express();

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

export default app;

4. Configuración de TypeScript (tsconfig.json)

Configuración completa para un proyecto TypeScript moderno:

tsconfig.json 
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "removeComments": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "noImplicitThis": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "moduleResolution": "node",
    "baseUrl": "./src",
    "allowSyntheticDefaultImports": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

5. Configuración de ESLint (.eslintrc.js)

Configuración para mantener la calidad del código:

.eslintrc.js 
module.exports = {
  parser: '@typescript-eslint/parser',
  parserOptions: {
    project: 'tsconfig.json',
    tsconfigRootDir: __dirname,
    sourceType: 'module',
  },
  plugins: ['@typescript-eslint'],
  extends: [
    'eslint:recommended',
    '@typescript-eslint/recommended',
    '@typescript-eslint/recommended-requiring-type-checking',
  ],
  root: true,
  env: {
    node: true,
    jest: true,
  },
  ignorePatterns: [
    '.eslintrc.js', 
    'dist/**/*'
  ],
  rules: {
    '@typescript-eslint/interface-name-prefix': 'off',
    '@typescript-eslint/explicit-function-return-type': 'off',
    '@typescript-eslint/explicit-module-boundary-types': 'off',
    '@typescript-eslint/no-explicit-any': 'warn',
    '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
    '@typescript-eslint/prefer-const': 'error',
  },
};

6. Configuración de Prettier (.prettierrc)

Configuración para formateo automático del código:

.prettierrc 
{
  "semi": true,
  "trailingComma": "es5",
  "singleQuote": true,
  "printWidth": 80,
  "tabWidth": 2,
  "useTabs": false,
  "bracketSpacing": true,
  "arrowParens": "always",
  "endOfLine": "lf"
}

7. Scripts del package.json

Agregamos el script para ejecutar la aplicación:

package.json 
"scripts": {
  "start": "ts-node src/app.ts"
}

Ejecutar Express

Para levantar el servidor, ejecutamos:

Terminal 
npm run start

Health Check EndPoint

Implementamos el primer endpoint de la API:

src/app.ts 
import express from 'express';

const app = express();

// Health check endpoint
app.get('/health', (req, res) => {
  res.status(200).json({
    success: true,
    message: 'API funcionando correctamente',
    timestamp: new Date().toISOString(),
    version: 'v1',
    environment: 'development',
  });
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

export default app;
¡Excelente! Ya tienes tu primer endpoint funcionando. Puedes probarlo visitando http://localhost:3000/health
